# Data Dictionary

This dictionary documents the Snowflake raw ingestion layer. Raw tables mirror the generated CSV files and add simple load metadata:

- `loaded_at`: load timestamp from Snowflake
- `source_file`: staged file name captured during `COPY INTO`

## RAW.PRODUCTS_RAW

Grain: One row per product.

Purpose: Product attributes used to describe sales and returns.

| Column | Type | Notes |
| --- | --- | --- |
| product_id | STRING | Synthetic product identifier. |
| product_name | STRING | Synthetic product display name. |
| product_category | STRING | Product category. |
| product_subcategory | STRING | Product subcategory. |
| brand | STRING | Synthetic brand. |
| unit_price | NUMBER(10,2) | Retail unit price. |
| cost | NUMBER(10,2) | Synthetic product cost. |
| active_flag | BOOLEAN | Whether the product is active. |
| loaded_at | TIMESTAMP_NTZ | Snowflake load timestamp. |
| source_file | STRING | File loaded from the stage. |

Notes: no product transformations are applied in the raw layer.

## RAW.STORES_RAW

Grain: One row per store.

Purpose: Store location and format attributes.

| Column | Type | Notes |
| --- | --- | --- |
| store_id | STRING | Synthetic store identifier. |
| store_name | STRING | Store display name. |
| city | STRING | Store city, primarily Ontario locations based in ON towns and cities. |
| province | STRING | Province abbreviation (just ON). |
| region | STRING | Regional grouping. |
| store_type | STRING | Store format such as mall, outlet, or flagship. |
| open_date | DATE | Store open date. |
| loaded_at | TIMESTAMP_NTZ | Snowflake load timestamp. |
| source_file | STRING | File loaded from the stage. |

Notes: Locations are synthetic and intended for portfolio analysis only.

## RAW.CUSTOMERS_RAW

Grain: One row per synthetic customer account.

Purpose: Non-identifying customer attributes for analyzing loyalty and returns behavior.

| Column | Type | Notes |
| --- | --- | --- |
| customer_id | STRING | Synthetic customer identifier. |
| loyalty_tier | STRING | Loyalty tier. |
| signup_date | DATE | Customer signup date. |
| city | STRING | Customer city. |
| province | STRING | Province abbreviation. |
| loaded_at | TIMESTAMP_NTZ | Snowflake load timestamp. |
| source_file | STRING | File loaded from the stage. |

Notes: This table intentionally excludes names, emails, phone numbers, street addresses, and other identifying customer fields.

## RAW.EMPLOYEES_RAW

Grain: One row per synthetic employee.

Purpose: Employee attributes used for sales and return processing analysis.

| Column | Type | Notes |
| --- | --- | --- |
| employee_id | STRING | Synthetic employee identifier. |
| store_id | STRING | Store where the employee is assigned. |
| role | STRING | Employee role. |
| hire_date | DATE | Employee hire date. |
| active_flag | BOOLEAN | Whether the employee is active. |
| loaded_at | TIMESTAMP_NTZ | Snowflake load timestamp. |
| source_file | STRING | File loaded from the stage. |

Notes: This table intentionally excludes employee names and private employee details.

## RAW.SALES_RAW

Grain: One row per sale line item.

Purpose: Raw sales transactions generated by the synthetic data generator.

| Column | Type | Notes |
| --- | --- | --- |
| sale_id | STRING | Synthetic sale line identifier. |
| transaction_id | STRING | Synthetic transaction identifier. |
| sale_date | DATE | Date of sale. |
| store_id | STRING | Store where the sale is associated. |
| customer_id | STRING | Synthetic customer identifier. |
| employee_id | STRING | Synthetic employee identifier. |
| product_id | STRING | Synthetic product identifier. |
| quantity | NUMBER(10,0) | Units sold. |
| unit_price | NUMBER(10,2) | Unit price at sale time. |
| discount_amount | NUMBER(10,2) | Discount applied to the sale line. |
| sale_amount | NUMBER(10,2) | Sale amount after discount. |
| payment_method | STRING | Payment method used. |
| sales_channel | STRING | Sales channel such as in_store, online, or curbside_pickup. |
| loaded_at | TIMESTAMP_NTZ | Snowflake load timestamp. |
| source_file | STRING | File loaded from the stage. |

Notes: the raw layer does not enforce foreign keys... yet! Relationship checks are expected in later QA/dbt layers.

## RAW.RETURNS_RAW

Grain: One row per return event linked to a sale line item.

Purpose: Raw product return events generated from valid synthetic sales.

| Column | Type | Notes |
| --- | --- | --- |
| return_id | STRING | Synthetic return identifier. |
| sale_id | STRING | Sale identifier referenced by the return. |
| transaction_id | STRING | Original sale transaction identifier. |
| return_date | DATE | Date of return. |
| store_id | STRING | Store where the return is associated. |
| customer_id | STRING | Synthetic customer identifier. |
| employee_id | STRING | Synthetic employee who processed the return. |
| product_id | STRING | Synthetic product identifier. |
| return_quantity | NUMBER(10,0) | Units returned. |
| return_amount | NUMBER(10,2) | Amount refunded or credited. |
| return_reason | STRING | Return reason. |
| return_channel | STRING | Return channel such as in_store, mail, or curbside. |
| refund_method | STRING | Refund method. |
| receipt_present | BOOLEAN | Whether a receipt was present. |
| item_condition | STRING | Returned item condition. |
| policy_exception_flag | BOOLEAN | True for unusual returns such as late, no receipt, damaged, or high-value cases. |
| loaded_at | TIMESTAMP_NTZ | Snowflake load timestamp. |
| source_file | STRING | File loaded from the stage. |

Note: Returns are generated to reference valid `sale_id` values however raw ingestion keeps data as loaded and does not apply transformations.
